## Use This File

- This is the final acceptance ledger, not the active agent task board.
- Section agents claim and update `TASK-###` work in `TASKS.md`; they do not add GREEN `A###`
  acceptance rows while work is still isolated in a branch/worktree.
- The final merge agent assigns new `A###` IDs after integration and validation, then updates this
  file, `WORKLOG.md`, and `naive_validation.md` together.
- If an unlanded branch contains a stale or colliding `A###` row, renumber or rewrite that row before
  landing. The landed/committed acceptance ID wins.
- Keep this file short. Put detailed run history in `WORKLOG.md`.
- Add only the current acceptance slice plus the last few relevant deltas.
